- `>` : creates/overwrites a file
- `>>` : appends to an existing file
- `|` (pipe) : passes output of one command as input to another
- `2>` : redirects standard error
- `&>` : redirects both stdout and stderr to a single file
- `/dev/null` : discards output or errors
- Combining `{ ... }` allows grouping multiple commands for redirection
- Verification files like `verification.txt` help confirm command correctness
